# Vibe Research

**版本 2.0.3 beta** · [English](README.md)

一套用 AI agent 做科学研究的 [Claude Code](https://claude.com/claude-code) skill。

它建立在一个前提上：**与有能力的 AI agent 协作，是管理问题，不是工具问题。** 当 agent 已经能跑实验、写代码、起草论文，瓶颈就移到了人这一侧：如何指挥、审阅、纠正一支才华出众的团队——它的目标和你不完全一致，而且它比你更清楚自己刚刚做了什么？管理学思考这个问题已经一百年了。这套框架把它的答案用在「一个人的 AI 实验室」上。

## 核心哲学

**交代意图，而不是指令。** 像优秀的指挥官那样委托：说清*为什么*、*做成什么样算完成*、*什么不能牺牲*，然后让 agent 自己选择手段。框架里的每条规则都带着它的理由，并且明确告诉 agent：当规则的字面与理由冲突时，去质疑规则，而不是盲目执行。不带理由的规则会产出荒谬的结果——比如把一个根本没人看的 checkpoint 文件夹重构得整整齐齐。

**图是接口。** 统计量是在你知道什么重要*之前*做的压缩，它会悄悄丢掉意外。所以每次实验必须以图收尾：完整的分布，而不是汇总数字。图也是唯一一条不经过 agent 自我汇报过滤的信息通道：它把「做得怎么样」和「agent 说做得怎么样」分开。人看图做判断，而这个判断刻意不自动化——任何交给优化器的固定指标，都会不再度量你原本想要的东西（古德哈特定律）。

**按恢复成本决策，而不是按 git。** 错误便宜可撤销的地方，agent 拥有完全自主权：草稿区、小实验、探索分支。恢复昂贵的地方必须先问——而「昂贵」以人来度量，不以仓库度量。重写论文的叙事是不可逆的，哪怕 git 能回滚：作者读过、想过一个版本之后，认知回不去了。

**按人读不读来组织。** 人要读的目录保持小而可扫读；人永远不读的目录（checkpoint、缓存）没有任何整洁要求。框架里的每一条简洁性约束，都以「真的存在一个读者」为前提。

**约束是发现出来的，不是设计出来的。** 在看到 agent 做错之前，你不知道自己不希望它做什么。所以框架不假装有一本完备的规则书，而是把「捕获」做得很便宜：每次纠正都变成一条规则（规则 + 理由 + 适用条件），写进项目自己的 CLAUDE.md，像伤疤组织一样生长。skill 存放跨项目通用的东西；CLAUDE.md 存放这个项目用代价换来的教训。记录死路时要谦卑：区分*根本走不通*和*只是还没探索够*——今天的死路，有时以后会打开。

**产出里没有对话。** 论文、代码、图面向的读者从没见过你和 agent 的对话。所以 agent 绝不能在产出里回应用户（「我们做了 B 但没做 A」），也绝不能和已删除的旧版本争论。解释都放在对话里，说多长都行；产出读起来永远像一次写成的。

**判断留在人手里。** 永远不委托的东西：定义研究问题、看图评判质量、决定何时转向或放弃、以及品味。凡是能流程化、能便宜验证、能便宜恢复的都交出去；剩下的就是研究者的工作——它不会缩到零。

## Skill 一览

| Skill | 做什么 |
|-------|--------|
| [`vibe-research`](skills/vibe-research/) | 核心：项目初始化与迁移、什么时候用哪个 skill、merge 验收。自带项目模板（每个目录一份 CLAUDE.md）。 |
| [`explore`](skills/explore/) | 探索循环：开分支、小步快跑的实验、每一步出图、如实汇报，方向由人决定。 |
| [`vibe-optimize`](skills/vibe-optimize/) | 大规模参数搜索（贝叶斯 / 演化），带防刷指标的护栏：图由人评分、汇报机制而不只是分数、搜索与正式结果严格隔离。 |
| [`elegant-coding`](skills/elegant-coding/) | 科研代码最佳实践：直接的实现、fail-fast、严格的尺寸限制、实验与画图分离。 |
| [`paper-writing`](skills/paper-writing/) | 论文规范与工作流：双语 arXiv LaTeX 模板、散文纪律、禁元写作、引用必须核实、编译通过才算完成。 |
| [`my-scientific-taste`](skills/my-scientific-taste/) | 一个名字固定的占位符。个人品味——合格科学家之间真正有分歧的判断——住在这里。把内容换成你自己的；其他 skill 按这个名字来参考它。 |

画图风格是独立的 skill：[nature-style](https://github.com/Zhangyanbo/nature-style-skill)。

三层结构，一句话说完：**对所有人成立的进 skill；对你成立的进 `my-scientific-taste`；对这个项目成立的进项目的 CLAUDE.md。**

## 安装

把这段话直接粘贴给你的 agent（Claude Code 或任何 coding agent）：

> 安装 Vibe Research skills：克隆 https://github.com/Zhangyanbo/vibe-research-skill ，把 `skills/` 下的每个文件夹拷贝到我的 skills 目录（Claude Code 是 `~/.claude/skills/`）。如果已装有旧版本就替换掉——但如果我已经有自己的 `my-scientific-taste`，保留我的。装完告诉我装了什么。

然后说一句类似 *「帮我 init 一个关于……的 vibe research 项目」*。agent 会问你研究问题、论文阅读语言、已有代码在哪，然后搭好项目。

正式使用之前，打开 `skills/my-scientific-taste/SKILL.md`，把内容换成你自己的品味。那个文件本来就该是你的。
